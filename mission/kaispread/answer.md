## 🔰 사전 미션
### 1. 컨테이너 기술이란 무엇입니까?
```
컨테이너는 애플리케이션과 필요한 모든 파일을 하나의 런타임 환경으로 묶는 데 사용하는 기술입니다.
기존 호스트형 가상화에서는 Guest OS 구동에 필요한 물리적인 자원을 소프트웨어로 구현하여 사용합니다. 
따라서, 실질적으로 완전히 다른 OS와 동등한 것으로 아무 운영체제나 설치할 수 있고 완전히 독립적으로 실행됩니다. 
반면, Docker와 같은 컨테이너 기술은 Host OS 위에 컨테이너 엔진을 통해 동작하므로 기존의 가상화 기술보다 훨씬 가볍게 동작합니다.
하지만, 운영체제의 일부 기능을 Host OS에 의존하기 때문에 아무 운영체제나 설치할 수 있는 가상화보다 제약사항이 있습니다.
``` 

### 2. 도커란 무엇입니까? 
```
컨테이너 기술을 사용하여 애플리케이션 실행 환경을 구축 및 운용하기 위한 오픈 소스 플랫폼입니다.
애플리케이션 실행에 필요한 환경을 하나의 이미지로 정의하고, 이미지를 사용해 다양한 환경에서 사용이 가능하다는 장점이 있습니다.
```

### 3. 도커 파일, 도커 이미지, 도커 컨테이너의 개념은 무엇이고, 서로 어떤 관계입니까?
```
- 도커 파일 : Docker 에서 Docker image를 생성하기 위한 용도로 작성하는 파일입니다. 이미지에 대한 정보를 기술해둔 하나의 템플릿 역할을 합니다.
- 도커 이미지 : Docker image는 애플리케이션 구동에 필요한 프로그램, 소스 코드, 라이브러리, 실행 파일들이 묶인 파일입니다. 
               즉, 컨테이너 실행에 필요한 모든 파일과 설정 값을 지닌 것으로, 더 이상의 추가 작업이 필요하지 않은 상태의 파일을 말합니다.
               이러한 이미지들은 Layer 개념에 의해 확장될 수 있습니다.
- 도커 컨테이너 : 도커 이미지가 구동되는 런타임 환경입니다. 도커 컨테이너는 Linux 환경에서 구동되며 여러개의 컨테이너를 Docker 엔진 위에 띄울 수 있고
               여러개의 컨테이너는 각각 돌깁적으로 실행됩니다. 이 컨테이너들은 커널 공간과 Host OS 자원을 공유합니다.

결국 Docker file을 통해 image를 만들고, 생성된 image를 Docker container에서 실행시킴으로써 애플리케이션을 컨테이너 환경에 구동시킬 수 있습니다. 
```
